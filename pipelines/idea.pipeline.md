# Idea Pipeline

## Purpose

Turn a raw idea into a project hypothesis that can be validated.

## Inputs

- User idea.
- Product context in `docs/product/`.
- Existing architecture constraints.

## Steps

1. Capture the problem, target users, value, and non-goals.
2. Identify assumptions and unknowns.
3. Map affected modules or missing modules.
4. Create validation questions.
5. Add tasks for unresolved decisions.

## Outputs

- Product hypothesis.
- Open decision tasks.
- Candidate module/API impacts.

## Dependencies

- Feeds `validation.pipeline.md`.
- May trigger `architecture.pipeline.md` when the idea changes system boundaries.
