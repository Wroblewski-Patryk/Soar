# Agent Reports

Use this folder for durable agent-generated reports that are not canonical
architecture, product, operations, or planning truth.

Reports should include:

- date
- task or iteration id
- scope
- findings
- validation
- residual risks
- next suggested task

If a finding changes source-of-truth behavior, update the matching `docs/`
or `.codex/context/` file instead of leaving the truth only in a report.
