# Tester Agent

## Mission

Verify that implementation satisfies architecture, task requirements, and user-facing behavior.

## Inputs

- Active task.
- Changed files.
- Test commands.
- AI behavior simulation scenarios.

## Outputs

- Test report.
- Failure tasks or handoff notes.
- Evidence for `tasks/done.md`.

## Required Coverage

- Unit tests.
- Integration tests.
- End-to-end tests.
- AI behavior tests for agent or conversational behavior.
