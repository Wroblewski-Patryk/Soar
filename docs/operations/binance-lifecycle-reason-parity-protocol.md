# Binance Side-by-Side Lifecycle Reason Parity Protocol

Date: `2026-03-29`  
Owner: `QA / Operator`

## Goal
- Verify that close reasons generated by CryptoSparrow match chart reality for the same market window and strategy setup.
- Validate parity for:
  - `take_profit` (`TP`)
  - `trailing_take_profit` (`TTP`)
  - `stop_loss` (`SL`)
  - `trailing_stop` (`TSL`)
- Confirm that the same lifecycle interpretation is preserved for:
  - `BACKTEST` report
  - `PAPER` runtime behavior
  - `LIVE` runtime decision core (execution side-effects excluded from this protocol)

## Preconditions
- App stack running locally (`api`, `web`, required workers, db/redis).
- Binance chart access for tested symbols.
- One fixed strategy with explicit close config (`basic` or `advanced`) and clear thresholds.
- One fixed market group (recommended: 3 symbols).
- Fixed interval and fixed date window.

## Freeze Test Profile (Before Run)
Record and freeze:
- Strategy id + exported JSON config
- Market group id + symbol list
- Market type (`SPOT` or `FUTURES`)
- Timeframe
- Date window
- Backtest `maxCandles`
- Leverage and margin mode
- Initial balance (for backtest/paper)

Any change means a new run id and a new protocol pass.

## Execution Steps
1. Open Binance charts for all symbols in the frozen set.
2. Configure each chart with identical timeframe and indicators used by strategy.
3. Run backtest in CryptoSparrow for frozen profile.
4. Open report tabs:
   - `Summary`
   - `Markets`
   - `Trades`
   - `Raw`
5. For each symbol, capture lifecycle events in order:
   - entry
   - DCA events (if enabled)
   - close reason (`TP` / `TTP` / `SL` / `TSL` / `LIQ`)
6. Compare each close event timestamp/candle with Binance chart context:
   - threshold reached (TP/SL) or
   - arm + retrace logic reached (TTP/TSL).
7. Confirm no impossible sequence appears (examples):
   - close reason without open position
   - overlapping positions on the same symbol
   - opposite-side flip before close
8. For `PAPER` mode, replay the same configuration and verify close-reason sequence is identical for overlapping candles.

## PASS Criteria
- For every tested symbol, close reasons in CryptoSparrow are explainable from Binance candles/indicators.
- Event order is consistent with lifecycle contract:
  - DCA first
  - then close protection logic
  - no no-flip violations
- No unexplained mismatch spikes in diagnostics (`mismatchCount` + `mismatchSamples` coherent).
- Close-reason sequence for paper run matches backtest sequence for shared candle range.

## FAIL Criteria
- Any close reason cannot be justified by chart data/config.
- Sequence violates lifecycle order or one-position-per-symbol rule.
- Backtest/paper disagree on close-reason sequence for same setup.
- Diagnostics show repeated unexplained mismatches.

## Evidence Bundle (Required)
- Backtest run id
- Strategy id + frozen config snapshot
- Market group id + symbols
- Binance screenshots per symbol (indicator overlays visible)
- CryptoSparrow markets screenshots per symbol (event markers + stats card)
- Exported close-reason sequence per symbol (`Trades`/`Raw` evidence)
- Verdict:
  - `PASS`
  - or `FAIL` with concrete reason and first failing symbol/candle
