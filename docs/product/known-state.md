## Mandatory Scope Boundaries (Must Not Regress For Go-To-Market)

- Exchange scope remains Binance Spot + Binance Futures only (`docs/product/known-limits.md`).
- Strategy model remains flat rule model in MVP (`docs/product/product.md`, `docs/product/known-limits.md`).
- No full user-facing manual trade ticket panel in dashboard for this MVP wave (`docs/product/product.md`).
- No portfolio-level AI auto-optimizer in MVP (`docs/product/product.md`).
- Localization supported at least EN/PL/pt-PT for core flows (`docs/product/localization.md`).

## Compliance / Legal Go-to-Market Acceptance Criteria

Use this list as the legal/market gate for release review:

1. **User risk governance**
   - [ ] `LIVE` mode is explicit opt-in per bot and requires consent text versioning.
   - [ ] Consent text in UI matches `docs/security/mvp-risk-consent-text.md` (EN/PL where relevant).
   - [ ] Consent action is persisted in audit log (`live_consent.accepted`) with `consentTextVersion`.

2. **Non-advisory positioning**
   - [ ] Public product surfaces never imply guaranteed returns or financial advice.
   - [ ] Marketing and onboarding copy preserves the "not financial advice" positioning.

3. **Trading safety and kill-switch posture**
   - [ ] LIVE operations have explicit kill-switch/emergency-stop checks for risky actions.
   - [ ] High-risk actions are protected by ownership checks, consent checks, and mode checks.
   - [ ] Paper and LIVE paths remain logically separated when required by user profile and mode.

4. **Secret and account security**
   - [ ] Exchange key create/rotate/revoke flows require ownership controls and never return secrets in plaintext.
   - [ ] Redis-backed rate limiting is confirmed for high-risk endpoint groups.
   - [ ] Authentication/session expiry and lockout behavior follows security policy.

5. **Evidence + support posture**
   - [ ] `docs/operations/mvp-release-checklist.md` readiness items are fully checked before launch.
   - [ ] Runbook references are present and current:
     - `docs/operations/v1-ops-runbook.md`
     - `docs/operations/mvp-ops-runbook.md`
     - `docs/security/api-key-lifecycle-policy.md`
   - [ ] No open "Known Limits" items are promoted into active promise surfaces.
