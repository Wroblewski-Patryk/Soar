# Modules Documentation

Use one file per module in this folder.

## Role of This Folder
`docs/modules/` explains:
- where the code lives,
- which API routes or UI surfaces a module owns,
- which dependencies it uses,
- which tests verify it.

It does not replace `docs/architecture/` as the canonical source of runtime behavior.

## Canonical File
- [System modules](./system-modules.md) contains the repository-wide module map.

## Coverage Audits
- Historical coverage audits live in `history/audits/`.

## Coverage Index
- [Module doc status index](./module-doc-status-index.md) tracks deep-dive status for each active API and web module.
- [Module registry](./module-registry.md) connects module docs to routes, pipelines, data, and tests.

## Deep-Dive Template
- [Module deep-dive template](./module-deep-dive-template.md) is the canonical template for new module documentation.
- Every new deep-dive should be created from this template and keep the full mandatory checklist.
