# Documentation Maps

Use these maps as human and agent entrypoints. They intentionally link to a
small set of high-signal documents instead of every historical proof file.
Guard note: history is evidence, not active owner.

## Maps

| Map | Purpose |
| --- | --- |
| [Product Map](./product-map.md) | Product intent, scope, roadmap, limits, and glossary. |
| [Architecture Map](./architecture-map.md) | Runtime source of truth, system topology, contracts, and module map. |
| [Runtime Map](./runtime-map.md) | Trading lifecycle, signal flow, exchange safety, bot runtime, and state parity. |
| [Release And Operations Map](./release-ops-map.md) | Deploy, rollback, smoke, SLO, release gate, and evidence history. |
| [Agent Work Map](./agent-work-map.md) | Coordinator workflow, active state, task board, quality gates, and history lookup. |

## Obsidian Tip

In graph view, filter out `history/` when you want the current system map.
Turn it back on when you need audit trail or proof lineage.
