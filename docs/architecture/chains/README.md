# Function Chains

Last updated: YYYY-MM-DD

Function chains connect ideas and user/system actions to the implementation
layers that make them real.

Use `chains.csv` as the registry. Add optional human-readable chain docs in
this folder when a chain is large or high risk.

Every chain should answer:

- what triggers it;
- which UI/API/service/data/worker/agent/doc/test nodes participate;
- what proof exists;
- what proof is missing;
- what next task increases confidence.
